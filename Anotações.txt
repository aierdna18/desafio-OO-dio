Paradigma: Modelo, exemplo a ser seguido
Programação: Desenvolvimento de uma aplicação
Orientada: Aquilo que é regido, segue uma orientação
Objeto: Um molde que representa uma determinada classe.

Paradigma de Programação Orientado a Objetos: É um modelo de desenvolvimentto de aplicações, sendo este modelo construído acerca do conceito de objetos.

Linguagem de programação: É uma ligueagem formal que , atraves de uma serie de instruções, permite que um programador escreva um conjunto de ordens, ações consecutivas, dados e algoritmos para criar programas. Podem ser classificadas:
  Niveis de abstração(baixo, médio alto)
  Paradigma de programação(estruturada, orientada a objeto)
  Arquitetura de aplicação(de acordo com arquitetura é escolhida a Linguagem)
  Tipo de execução(interpretada, compilada,híbrida)

Paradigma de Linguagem de Programação: é um conjunto de características que podem ser utilizadas para categorizar determinado grupo de lingueagens. Um paradima pode oferecer técnicas apropriadas para uma aplicação específica.
  Paradigmas Principais e seus Subparadigmas:
    Paradigma Imperativo: Estruturado, Concorrente, OO 
    Paradigma Declarativo: Funcional, Lógico

Dominío da aplicação: é a camada do codigo onde ficam as classes que vão descrever o problema.
